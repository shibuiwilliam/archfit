# P2.SPC.001 — API boundary has a machine-readable contract

The repo looks like it serves an API but has no OpenAPI, Protobuf, or GraphQL
schema file.

## Decision tree

1. **Is this repo actually a service with an API?**
   - No → suppress with an ignore entry. Ask the user to confirm.
   - Yes → continue.

2. **Which API style does the service use?**
   - REST/HTTP → create `api/openapi.yaml` with an OpenAPI 3.x skeleton.
   - gRPC → create `proto/<service>.proto` with service definitions.
   - GraphQL → create `graphql/schema.graphql` with type definitions.

3. **Generate the skeleton automatically?**
   - Ask the user: "I detected service indicators at {paths}. Shall I create
     an OpenAPI/Protobuf/GraphQL skeleton?" Do NOT create without confirmation.

## Minimal fix

```yaml
# api/openapi.yaml
openapi: "3.0.3"
info:
  title: My Service API
  version: "0.1.0"
paths: {}
```

Or for gRPC:

```protobuf
// proto/service.proto
syntax = "proto3";
package myservice;

service MyService {
  // Define RPCs here.
}
```
