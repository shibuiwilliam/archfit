# Remediation: P3.EXP.001 — Configuration not documented

**Summary**: The repo has hidden configuration files without corresponding documentation. archfit detects: `.env` files (requires `.env.example`), Spring Boot `application-*.yml` profiles (requires `config/README.md`), Terraform `*.tfvars` (requires `terraform.tfvars.example`), and Rails `config/environments/` (requires `config/README.md`).

## Decision tree

1. **Check the `config_type` in finding evidence** to know which mechanism was detected:
   - `dotenv` → create `.env.example`
   - `spring-boot` → create `config/README.md` documenting profile differences
   - `terraform` → create `terraform.tfvars.example`
   - `rails` → create `config/README.md` documenting environment differences

2. **Is the config genuinely used?**
   - No (leftover / generated) → suggest `.archfit.yaml` suppression with a rationale.
   - Yes → continue.

3. **Create the documentation file** following the snippets below.

4. **Re-scan** to confirm.
