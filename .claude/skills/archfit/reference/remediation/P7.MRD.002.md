# Remediation: P7.MRD.002 — CHANGELOG.md missing

**Summary**: No `CHANGELOG.md` (or common variant) at the repository root.

## Decision tree

1. **Is the repo a shared agent-tool (consumed by others) or purely internal?**
   - Internal-only bootstrap script → suppress with rationale.
   - Shared → continue.

2. **Do any previous releases exist (git tags, `version.go`, `package.json` version)?**
   - Yes → seed the CHANGELOG with a retroactive summary. **Ask the user** for the prior notable changes; don't fabricate them.
   - No → start with `[Unreleased]` only.

3. **Create `CHANGELOG.md`** following Keep a Changelog.

4. **Re-scan** to confirm.

## Template

```markdown
# Changelog

All notable changes to this project will be documented in this file.
This file follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/)
and this project adheres to [SemVer](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- …

### Changed
- …

## [0.1.0] - YYYY-MM-DD

### Added
- Initial release.
```

## Anti-patterns

- Listing commit messages verbatim — summarize, don't dump `git log`.
- Breaking changes buried under "Changed" — promote them to their own section.
