# Remediation: P5.AGG.004 — High-risk paths not protected by CODEOWNERS

**Summary**: The repository has directories with auth, secret, migration, or deployment files but no CODEOWNERS file to enforce mandatory review.

## Decision tree

1. **Does the repo have multiple contributors?**
   - No (solo project) → suppress with rationale and expiry date.
   - Yes → continue.

2. **Does the hosting platform support CODEOWNERS?**
   - GitHub, GitLab, Bitbucket → continue.
   - Other → suppress with rationale noting the alternative mechanism.

3. **Create `.github/CODEOWNERS`** with entries for each high-risk category listed in the finding evidence.

4. **Re-scan** to confirm the finding disappears.

## Example CODEOWNERS

```
# Auth
/src/auth/       @myorg/security-team
/internal/auth/  @myorg/security-team

# Migrations
/migrations/     @myorg/dba-team

# Infrastructure
/infra/          @myorg/platform-team
/deploy/         @myorg/platform-team
```

## When to suppress

```yaml
ignore:
  - rule: P5.AGG.004
    reason: "Solo maintainer project — no team for CODEOWNERS"
    expires: 2027-06-01
```
