# Remediation: P1.LOC.004 — Wide change sets

**Summary**: Typical commits touch more than 8 files, indicating poor change isolation.

## Decision tree

1. **Is this a temporary spike (e.g., active refactor)?**
   - Yes → suppress with rationale and expiry date.
   - No → continue.

2. **Are cross-cutting changes caused by coupling?**
   - Identify the most common co-changed file pairs.
   - Extract shared logic to reduce coupling.
   - **Ask the user** before restructuring modules.

3. **Re-scan** after the refactor to verify improvement.
