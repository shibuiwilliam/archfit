# Remediation: P1.LOC.001 — Agent-facing docs at repo root

**Summary**: The root of the repository has no `CLAUDE.md` or `AGENTS.md`.

## Decision tree

1. **Is this a repository the user actively develops in?**
   - No → Suggest `.archfit.yaml` suppression with a `reason` and `expires`. Do not create a doc in someone else's read-only repo.
   - Yes → continue.

2. **Does the user already have a convention for agent-facing docs (e.g., `agent_docs/` directory, `README.md` with an "Agents" section)?**
   - Yes → **Ask the user** whether to add `CLAUDE.md` as a pointer to the existing docs, or to suppress P1.LOC.001 with a rationale.
   - No → continue.

3. **Propose a new `CLAUDE.md`** with this minimum outline, tailored to the repo:
   - What the repo is (one sentence).
   - Top-level layout.
   - Fast verification loop (the command from P4.VER.001).
   - Conventions the agent must follow.
   - When this file and reality disagree, ask before changing either.

4. **Apply, then re-scan** with `archfit scan --json .` to confirm the finding disappears.
