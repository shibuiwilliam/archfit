# Remediation: P2.SPC.002 — DB migrations not bidirectional

**Summary**: A migrations directory has SQL files in only one direction (up or down), missing the reverse pair.

## Decision tree

1. **Are the migrations legacy (predating the up/down convention)?**
   - Yes -> suppress with `.archfit.yaml` ignore entry and a plan to retrofit.
   - No -> continue.

2. **Create the missing direction file** for each unpaired migration:
   - `001_create_users.up.sql` needs `001_create_users.down.sql`.
   - The down migration should reverse the up (e.g., `DROP TABLE`).

3. **Re-scan** to confirm the finding is gone.

## Example

```sql
-- 001_create_users.down.sql
DROP TABLE IF EXISTS users;
```
