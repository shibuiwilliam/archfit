# P5.AGG.002 — Repository runs a secret scanner in CI

CI is configured but no secret scanner was detected in the pipeline.

## Decision tree

1. **Does the repo already use a secret scanner?**
   - Yes, but it's platform-native (e.g., GitHub Advanced Security) →
     suppress with an ignore entry. Ask the user to confirm.
   - Yes, but archfit didn't detect it → file an issue.
   - No → continue.

2. **Which CI platform?**
   - GitHub Actions → add `gitleaks/gitleaks-action` workflow.
   - GitLab CI → add `gitleaks` as a CI job.
   - Other → add `gitleaks` or `trufflehog` as a script step.

3. **Ask the user** before creating the workflow file.

## Minimal fix (GitHub Actions)

```yaml
# .github/workflows/gitleaks.yml
name: gitleaks
on: [push, pull_request]
jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: gitleaks/gitleaks-action@v2
```
