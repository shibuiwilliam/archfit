# P6.REV.002 — Deploying repository uses a feature-flag mechanism

Repository deploys but has no feature-flag mechanism. Changes cannot be
toggled off without a full rollback.

## Decision tree

1. **Does the repo deploy a runtime service?**
   - No (static site, IaC-only, library) → suppress with ignore entry.
     Ask the user to confirm.
   - Yes → continue.

2. **Does the team already use a flag service not detected?**
   - Yes → file an issue to add the keyword. Suppress temporarily.
   - No → continue.

3. **Suggest adding a flag library.**
   - Ask: "Your repo deploys but has no feature-flag mechanism. Would you
     like me to add OpenFeature (vendor-neutral) or a local config toggle?"
   - Do NOT add without confirmation.

## Minimal fix (local toggle)

```go
// internal/featureflags/flags.go
package featureflags

var Flags = map[string]bool{
    "new-checkout-flow": false,
}

func Enabled(name string) bool {
    return Flags[name]
}
```
