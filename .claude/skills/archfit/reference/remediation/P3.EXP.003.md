# Remediation: P3.EXP.003 — Reflection/metaprogramming density bounded

**Summary**: Too many files import the `reflect` package. Replace reflection-based dispatch with type switches, generics, or code generation.

## Decision tree

1. **Check the finding evidence** for the list of files importing reflect.
2. **Is reflect used for serialization (JSON, XML, etc.)?**
   - Yes -> this is expected; consider suppressing for those paths.
3. **Is reflect used for runtime dispatch or factory patterns?**
   - Yes -> replace with explicit type switches or generics.
4. **Is reflect used in test helpers?**
   - Yes -> acceptable; suppress for test paths.
5. **Re-scan** to confirm the count drops to the threshold or below.

## Example fix

```go
// Before: reflect-based factory
func create(name string) any {
    t := registry[name]
    return reflect.New(t).Interface()
}

// After: explicit factory
func create(name string) any {
    switch name {
    case "foo":
        return &Foo{}
    case "bar":
        return &Bar{}
    }
    return nil
}
```
