# Remediation: P4.VER.001 — No fast verification entrypoint

**Summary**: The repo has no recognized build/project file at the root. archfit checks for: `Makefile`, `justfile`, `Taskfile.yml`, `package.json`, `pyproject.toml`, `Cargo.toml`, `go.mod`, `pom.xml`, `build.gradle(.kts)`, `settings.gradle(.kts)`, `Gemfile`, `Rakefile`, `composer.json`, `mix.exs`, `build.sbt`, `CMakeLists.txt`, `deno.json`, `Earthfile`, `BUILD.bazel`, `meson.build`.

## Decision tree

1. **Does the repo have code that can be verified (tests, a build, a type-checker)?**
   - No (pure docs or content repo) → suggest `.archfit.yaml` suppression with a rationale.
   - Yes → continue.

2. **What is the language and build tool?**
   - Ask the user if the answer is ambiguous. Do not guess between `Makefile` and `Taskfile.yml` for the user.

3. **Create a minimal entrypoint** that runs the *actual* fast checks. Do not stub `test: @echo ok` — that makes the signal worse, not better.

4. **Re-scan** to confirm.

## Language-specific snippets

### Go

```makefile
.PHONY: test lint
test:
	go test -race -count=1 ./...
lint:
	go vet ./...
```

### Node / TS

```json
{
  "scripts": {
    "test": "vitest run",
    "lint": "eslint ."
  }
}
```

### Python

```toml
# pyproject.toml
[tool.poe.tasks]
test = "pytest -q"
lint = "ruff check ."
```

### Java (Maven)

Already recognized via `pom.xml`. Ensure `mvn test` works.

### Java (Gradle)

Already recognized via `build.gradle` / `build.gradle.kts`. Ensure `./gradlew test` works.

### Ruby

Already recognized via `Gemfile` or `Rakefile`. Ensure `bundle exec rspec` or `rake test` works.

### PHP

Already recognized via `composer.json`. Ensure `composer test` works.
