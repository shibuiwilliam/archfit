# Remediation: P3.EXP.002 — init() cross-package registration

**Summary**: Go `init()` functions register handlers, drivers, or middleware in other packages. This creates invisible side effects that agents cannot trace by reading code.

## Decision tree

1. **Is the init() doing cross-package registration?**
   - Check `cross_pkg_calls` in finding evidence for the specific calls.
   - If `--depth=standard` was used, re-run with `--depth=deep` for details.

2. **Is the registration required at import time?**
   - Yes (e.g., database driver for `database/sql`) → consider suppressing
     with a rationale if there is no alternative API.
   - No → continue to step 3.

3. **Move the registration to an explicit function.**
   - Create a `Register()` or `Setup()` function in the same package.
   - Call it from `main()` or a wiring function.
   - Remove the `init()` function.

4. **Re-scan** to confirm the finding disappears.

## Example fix

```go
// Before: implicit registration in init()
package mypkg

import "net/http"

func init() {
    http.HandleFunc("/api", apiHandler)
}

// After: explicit registration
package mypkg

import "net/http"

func RegisterRoutes(mux *http.ServeMux) {
    mux.HandleFunc("/api", apiHandler)
}
```

## When to suppress

```yaml
ignore:
  - rule: P3.EXP.002
    paths: ["pkg/drivers/**"]
    reason: "database/sql requires init()-time driver registration"
    expires: 2027-06-01
```
