# Remediation: P7.MRD.003 — No docs/adr/ for an agent-tool with a CLI

**Summary**: The repo has a `cmd/` binary but no ADR directory.

## Decision tree

1. **Has the project made irreversible design decisions worth recording?**
   - Early prototype, nothing load-bearing yet → suppress with a short `expires`.
   - Mature code with real decisions → continue.

2. **Create `docs/adr/` and seed ADR 0001** — the architecture overview. It names the top-level boundaries and the decisions behind them. **Ask the user** for the list of decisions to record if you are uncertain; don't manufacture rationales.

3. **Re-scan** to confirm.

## Minimum first ADR

```markdown
---
id: 0001
title: Architecture overview
status: accepted
date: YYYY-MM-DD
tags: [architecture, foundation]
---

# ADR 0001 — Architecture overview

## Context

<one paragraph: what drove the decision, what was considered>

## Decision

<one paragraph per load-bearing choice>

## Consequences

**Positive**: <…>
**Negative**: <…>

## Status

Accepted.
```

## Anti-patterns

- Writing ADRs for decisions nobody made ("we use Go"). ADRs are for moments of genuine tension between alternatives.
- Copying the code as the ADR. The code answers "what"; the ADR answers "why this and not that".
