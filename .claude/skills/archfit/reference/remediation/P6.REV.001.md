# Remediation: P6.REV.001 — No rollback/deployment documentation

**Summary**: The repo has deployment artifacts (Dockerfile, k8s configs, CI workflows, etc.) but no runbook or deployment documentation.

## Decision tree

1. **Does the repo actually deploy to production?**
   - No (deployment artifacts are for local dev only) → suggest `.archfit.yaml` suppression with a rationale.
   - Yes → continue.

2. **Does rollback documentation exist elsewhere?**
   - In a separate ops repo → ask the user whether to cross-reference or suppress.
   - Nowhere → create it.

3. **Create `docs/deployment.md`** with three sections: how to deploy, how to verify, how to roll back.
   - Ask the user for the deployment mechanism and rollback procedure.
   - Do not invent deployment procedures — ask.

4. **Re-scan** to confirm.

## Snippet

```markdown
# Deployment

## How to deploy
[Describe the deployment trigger and process]

## How to verify
[Health check, dashboard link, smoke test]

## How to roll back
[Exact commands or steps to revert to the previous version]

## Escalation
[Who to contact if rollback fails]
```

## Checklist

- [ ] The rollback procedure has been tested at least once.
- [ ] Dashboard/monitoring links are included (not just prose).
- [ ] The document is referenced from CLAUDE.md / AGENTS.md.
