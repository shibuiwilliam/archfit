# P2.SPC.002 — DB migrations are bidirectional

| Field | Value |
|---|---|
| Principle | P2 (Spec-first) |
| Severity | `warn` |
| Evidence | `strong` |
| Stability | `experimental` |

## What it checks

If a `migrations/` (or similar) directory contains `.sql` files following
an up/down naming convention, both directions must be present. Files named
`*up*` must have corresponding `*down*` counterparts.

## Why it matters

Unidirectional migrations signal that rollback was not considered as part of
the schema design. Agents proposing schema changes need to know whether
rollback is expected, and a missing down migration is a strong signal that
the spec is incomplete.

## How to fix

For every `*.up.sql` file, create a corresponding `*.down.sql` that
reverses the change (e.g., `DROP TABLE` for a `CREATE TABLE`).

## Suppressing this rule

```yaml
ignore:
  - rule: P2.SPC.002
    reason: "Legacy migrations predate up/down convention"
    expires: 2026-12-31
```
