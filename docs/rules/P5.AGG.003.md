# P5.AGG.003 — Repository declares a risk-tier file

| Field | Value |
|---|---|
| Principle | P5 (Aggregation of Dangerous Capabilities) |
| Severity | `warn` |
| Evidence | `strong` |
| Stability | `experimental` |

## What it checks

The repository should declare which paths are high-risk via RISK_TIERS.md,
SECURITY.md, or a `risk_tiers` block in `.archfit.yaml`. Without a risk-tier
declaration, agents treat all paths as equally safe.

## How to fix

Create one of: RISK_TIERS.md, SECURITY.md (with path annotations), or add
`risk_tiers` to `.archfit.yaml`.

## Suppressing this rule

```yaml
ignore:
  - rule: P5.AGG.003
    reason: "Small project with no high-risk paths"
    expires: 2027-01-01
```
