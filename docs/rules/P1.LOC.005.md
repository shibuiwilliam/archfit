# P1.LOC.005 — High-risk paths declare INTENT.md

| Field | Value |
|---|---|
| Principle | P1 (Locality) |
| Severity | `warn` |
| Evidence | `strong` |
| Stability | `experimental` |

## What it checks

Directories whose name contains a high-risk keyword (`auth`, `secret`,
`migration`, `deploy`) must contain an `INTENT.md` file. Fixture and
testdata directories are excluded.

## Why it matters

High-risk directories carry elevated blast radius. A change in `src/auth/`
that misunderstands the authentication flow can introduce security
vulnerabilities. An `INTENT.md` gives agents the invariants they must
preserve before touching the directory.

## How to fix

Create an `INTENT.md` in each flagged directory. Include:

- What the directory does (one sentence).
- Invariants that must hold after any change.
- Links to relevant specs or ADRs.

## Suppressing this rule

```yaml
ignore:
  - rule: P1.LOC.005
    reason: "Auth module is trivial wrapper, no special invariants"
    expires: 2026-12-31
```
