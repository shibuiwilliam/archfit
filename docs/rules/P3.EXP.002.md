# P3.EXP.002 — No init() cross-package registration

| Field | Value |
|---|---|
| Principle | P3 (Shallow Explicitness) |
| Severity | `warn` |
| Evidence | `strong` |
| Stability | `experimental` |
| Applies to | Go |

## What it checks

This rule detects Go `init()` functions that register handlers, drivers,
middleware, or other side effects in packages they import. Common examples:

- `func init() { http.HandleFunc("/health", handler) }`
- `func init() { sql.Register("mydriver", &Driver{}) }`
- `func init() { prometheus.MustRegister(myMetric) }`

These registrations are invisible to an agent searching for where behavior
is wired. The agent greps for `HandleFunc` or `Register` in `main()` and
finds nothing — the registration happens at import time.

At `--depth=standard`, archfit detects the presence of `init()` functions.
At `--depth=deep`, it additionally extracts the cross-package calls inside
each `init()` body and reports them in the finding evidence.

## Why it matters

P3 (Shallow Explicitness) requires that behavior be traceable by reading,
not by running. `init()` side effects are the most common violation in Go
codebases. They create implicit coupling: importing a package changes
global state without any visible call at the import site.

For agents, this is adversarial. The agent's mental model of the program
is built by reading code and following call chains. `init()` breaks this
model by introducing effects that happen before `main()` runs.

## How to fix

Move registrations from `init()` to explicit calls in `main()` or a
dedicated wiring function:

```go
// Before (implicit)
package drivers

import "database/sql"

func init() {
    sql.Register("mydb", &Driver{})
}

// After (explicit)
package drivers

func Register() {
    sql.Register("mydb", &Driver{})
}

// In main.go:
func main() {
    drivers.Register()
    // ...
}
```

## When init() is acceptable

- `init()` that sets package-local state without calling into other
  packages (e.g., compiling a regexp) is not flagged at `--depth=deep`.
- Test helpers that use `init()` are excluded (test files are not parsed).

## Suppressing this rule

