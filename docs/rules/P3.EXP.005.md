# P3.EXP.005 — Global mutable state minimized

| Field | Value |
|---|---|
| Principle | P3 (Shallow Explicitness) |
| Severity | `info` |
| Evidence | `medium` |
| Stability | `experimental` |
| Applies to | Go |

## What it checks

This rule counts mutable package-level variables (`var` declarations at
package scope) across the entire repository. If the total exceeds the
threshold (default: 10), the rule fires. Blank identifiers (`_`) are
excluded.

## Why it matters

Package-level mutable variables are global state. Any function in the
package can read or write them without an explicit parameter, making data
flow invisible to agents reading the code. Keeping the count low
encourages dependency injection through struct fields and function
parameters, which are traceable by reading.

## How to fix

1. Convert read-only `var` declarations to `const` where possible.
2. Move mutable state into struct fields passed explicitly.
3. Use function parameters instead of package-level variables.

```go
// Before
var timeout = 30 * time.Second

// After
type Config struct {
    Timeout time.Duration
}
```

```yaml
    expires: 2027-01-01
```
