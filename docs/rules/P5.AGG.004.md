# P5.AGG.004 — High-risk paths are protected by CODEOWNERS

| Field | Value |
|---|---|
| Principle | P5 (Aggregation of Dangerous Capabilities) |
| Severity | `error` |
| Evidence | `strong` |
| Stability | `experimental` |

## What it checks

This rule fires when a repository contains directories whose path segments
match high-risk keywords **and** no CODEOWNERS file exists.

High-risk categories detected:

- **auth**: auth, authentication, authorization, login, oauth, session, jwt, rbac, acl, permission
- **secret**: secret, credential, token, apikey, encrypt, decrypt, cipher, crypto
- **migration**: migration, migrate, schema_change
- **deploy**: deploy, infra, terraform, cloudformation, pulumi, ansible

CODEOWNERS locations checked:

- `.github/CODEOWNERS`
- `CODEOWNERS` (repo root)
- `docs/CODEOWNERS`

## Why it matters

P5 asks whether dangerous capabilities are aggregated and guarded.
CODEOWNERS is the simplest enforceable guardrail: it requires mandatory
review from designated owners before changes to protected paths can merge.

Without CODEOWNERS, an agent (or a junior contributor) can modify auth
logic, migration scripts, or deployment configs without any domain expert
ever seeing the change. This is the highest-risk gap archfit measures,
which is why this rule carries `error` severity — the first in archfit.

## Why error severity

This is archfit's first `error`-severity rule. The bar for error is:

1. **Strong evidence** — we can definitively confirm the absence of a
   CODEOWNERS file. There is no false-positive ambiguity.
2. **High blast radius** — unprotected high-risk paths affect the security
   posture of the entire repository.
3. **Cheap to fix** — adding a CODEOWNERS file takes minutes.

## How to fix

Create a CODEOWNERS file at `.github/CODEOWNERS`:

```
# Require security team review for auth changes
/src/auth/          @myorg/security
/internal/auth/     @myorg/security

# Require DBA review for migrations
/migrations/        @myorg/dba

# Require platform team review for infra
/infra/             @myorg/platform
/deploy/            @myorg/platform
```

## When no high-risk paths exist

If the repository has no directories matching any high-risk keyword, the
rule does not fire — CODEOWNERS is not required for repos without dangerous
capabilities.

## Suppressing this rule

```yaml
ignore:
  - rule: P5.AGG.004
    reason: "Single-person project, no team to assign as owners"
    expires: 2027-01-01
```
