# P1.LOC.009 — Runbook per high-risk slice

| Field | Value |
|---|---|
| Principle | P1 (Locality) |
| Severity | `warn` |
| Evidence | `strong` |
| Stability | `experimental` |

## What it checks

If the repository contains deployment artifacts (Dockerfile, Terraform
files, `docs/deployment.md`, `deploy/`, `k8s/`, `helm/`, etc.), a
`RUNBOOK.md` must exist at the repo root or in `docs/`.

## Why it matters

Deployment artifacts without a runbook mean that operational procedures live
only in human memory. Agents proposing infrastructure changes cannot reason
about rollback, escalation, or blast radius without a written runbook.

## How to fix

Create a `RUNBOOK.md` at the repository root. Include at minimum:

- Deployment procedure (step by step).
- Rollback procedure.
- On-call escalation path.
- Known failure modes and their mitigations.

## Suppressing this rule

```yaml
ignore:
  - rule: P1.LOC.009
    reason: "Runbook lives in external wiki, linked from README"
    expires: 2026-12-31
```
