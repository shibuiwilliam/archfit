# P2.SPC.004 — ADRs use YAML frontmatter

| Field | Value |
|---|---|
| Principle | P2 (Spec-first) |
| Severity | `info` |
| Evidence | `strong` |
| Stability | `experimental` |

## What it checks

ADR files in `docs/adr/` should begin with YAML frontmatter containing
at minimum: `id`, `title`, `status`, `date`. This makes ADRs machine-
parseable so agents can query decision history programmatically.

## How to fix

Add YAML frontmatter to each ADR:

```yaml
---
id: "0001"
title: Use PostgreSQL for persistence
status: accepted
date: 2026-01-15
---
```

## Suppressing this rule

```yaml
ignore:
  - rule: P2.SPC.004
    reason: "Team uses a different ADR format"
    expires: 2027-01-01
```
